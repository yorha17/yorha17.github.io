webpack 学习

初始化环境

  1. 新建文件夹
  2. 在文件夹中打开命令窗口，执行npm init -y，生成 package.json 文件
  3. js 代码写在src文件夹下
  4. 安装 webpack

安装

安装 webpack、webpack-cli 两个包

-D即--save-deb，即开发依赖，代表此包仅开发时使用

运行脚本准备

  1. 修改package.json文件，在script对象中添加属性名为build,属性值为webpack，此处属性名可以任意规定，属性值为实际执行的命令

修改默认出口

默认出口即 webpack 打包后生成的 js 文件，官网中说明如下：

output 属性告诉 webpack 在哪里输出它所创建的 bundle，以及如何命名这些文件。主要输出文件的默认值是 ./dist/main.js，其他生成文件默认放置在 ./dist 文件夹中。
你可以通过在配置中指定一个 output 字段，来配置这些处理过程:

注意：出口文件路径必须使用绝对路径

webpack 初级使用

  1. 写代码，在 src 文件夹中按照配置的规则新建入口 js 文件并编辑代码
  2. 运行 webpack 脚本打包 js 代码，npm run build

默认生成的是main.js，这里是进行了配置的结果

html 文件部署

入门方式

在根目录下新建 public 文件夹，html 文件放在该文件夹中，js 文件打包完成后将 html 文件手动复制到 dist 文件夹中

自动部署方式

安装部署用插件 html-webpack-plugin

插件介绍：

HtmlWebpackPlugin 简化了 HTML 文件的创建，以便为你的 webpack 包提供服务。这对于那些文件名中包含哈希值，并且哈希值会随着每次编译而改变的 webpack 包特别有用。你可以让该插件为你生成一个 HTML 文件，使用 lodash 模板提供模板，或者使用你自己的 loader。

安装：

npm i html-webpack-plugin -D

加载器

加载器 css-loader:解析 css 代码
加载器 style-loader：把解析后的 css 代码插入到 DOM

步骤：

  1. 准备 css 文件引入到 src/main.js 中
  2. 下载 css-loader 和 style-loader 本地软件包
    npm i css-loader style-loader --save-dev
  3. 配置 webpack.config.js 让 webpack 拥有该加载器功能
